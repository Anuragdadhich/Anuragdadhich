> Serverless data pipeline analyzing **88.6MB** of real Spotify data — albums, artists, tracks

```
📦 albums.csv  (76.3 MB) ─┐
👤 artists.csv  (1.7 MB) ─┼──► S3 Data Lake ──► Glue ETL ──► Athena SQL ──► QuickSight
🎵 tracks.csv  (10.6 MB) ─┘                                                      │
                                                                                  ▼
                                                              📊 K-pop · Bar · Donut Charts
```

| 🔧 Service | 💡 What I used it for |
|---|---|
| 📦 Amazon S3 | Data lake — stored 3 raw CSV files (88.6 MB total) |
| ⚙️ AWS Glue 4.0 | Visual ETL — joined album + artist + track tables |
| 🔍 Amazon Athena | SQL on S3 files — no database server needed |
| 📊 Amazon QuickSight | Dashboards — bar, vertical bar, K-pop donut chart |
| 🔐 AWS IAM | Permissions — AmazonAthenaFullAccess policy |

**➡️ [View Project on GitHub](https://github.com/Anuragdadhich/spotify-aws-analytics-pipeline)**
